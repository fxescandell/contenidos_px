Procesamiento automático

Controla cómo el sistema reacciona cuando llegan archivos nuevos al hotfolder. Desactiva el watcher para procesar solo de forma manual o por API.

Cuando llega un archivo al hotfolder, el pipeline se ejecuta de forma automática sin intervención manual.
Revisa continuamente el hotfolder buscando archivos nuevos. Desactiva si solo quieres procesar a demanda.
Cada cuántos segundos revisa el hotfolder (requiere reiniciar la app).

Rendimiento y filtros

Ajusta el comportamiento del pipeline para evitar procesar archivos basura o sobrecargar el sistema.

Archivos más pequeños se ignoran automáticamente (.DS_Store, archivos vacíos, etc).
Cuántos lotes se pueden procesar al mismo tiempo. Más de 3 puede ralentizar el equipo.
Envía un mensaje a Telegram cuando un lote falla el procesamiento. Requiere Telegram configurado.

Identificación

Nombre visible en el panel y en los ficheros de exportación WordPress.
En desarrollo se muestran más logs detallados. En producción solo errores.

Diagnóstico del sistema

Información en tiempo real del servidor. Útil para verificar que todo funciona correctamente.
