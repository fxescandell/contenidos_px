Actúa como un arquitecto de software senior y desarrollador Python especializado en OCR, procesamiento documental, NLP aplicado a clasificación de contenidos, heurísticas editoriales y sistemas de confianza para pipelines automáticos.

Quiero que implementes la capa de extracción, OCR, agrupación, clasificación híbrida y scoring de confianza para una aplicación que automatiza la creación de contenidos en WordPress a partir de carpetas, PDFs, DOCX e imágenes.

No te centres todavía en UI bonita ni en WordPress final. En esta fase quiero resolver muy bien el corazón documental del sistema.

========================================
1. OBJETIVO DE ESTA FASE
========================================

Quiero un módulo robusto que haga lo siguiente:

1. analizar un lote de archivos que llega desde una carpeta caliente
2. identificar qué archivos pertenecen juntos a un mismo artículo candidato
3. extraer texto de DOCX, PDF digital, PDF escaneado e imágenes
4. limpiar y normalizar ese texto
5. detectar señales útiles dentro del contenido
6. clasificar automáticamente:
   - municipio
   - categoría/CPT
   - subtipo
7. calcular niveles de confianza por etapa
8. explicar por qué ha clasificado así
9. decidir si el contenido puede seguir automático o debe ir a revisión

Quiero una implementación muy modular, testeable y trazable.

========================================
2. CONTEXTO DEL NEGOCIO
========================================

El contenido que llega pertenece a un único WordPress editorial.

Municipios principales:
- Cerdanya
- Maresme
- Berguedà
- General

Categorías/CPT:
- agenda
- noticies
- esports
- turisme actiu
- nens i joves
- cultura
- gastronomia
- consells
- entrevistes

Casos especiales:
- cultura puede ser cultura general o libro
- gastronomia puede ser general o receta
- agenda puede ser agenda simple o agenda con varias actividades
- algunos artículos llegan solo como cartel en imagen
- algunos lotes traen un DOCX principal con imágenes
- algunos PDFs son digitales y otros escaneados
- a veces la carpeta ya dice claramente la categoría
- a veces el contenido contradice la carpeta por error humano

========================================
3. ENTRADAS REALES
========================================

Los lotes pueden tener esta estructura:

- carpeta raíz tipo "cerdanya 320"
- subcarpetas tipo agenda / cultura / gastronomia
- dentro puede haber:
  - carpeta por artículo
  - DOCX + imágenes
  - PDF + imágenes
  - solo cartel JPG o PNG
  - imágenes mezcladas
  - PDFs individuales

Quiero que implementes lógica para agrupar por artículo candidato con distintos niveles de confianza.

========================================
4. RESULTADO ESPERADO
========================================

Cada artículo candidato debe acabar con:

- lista de archivos asociados
- texto extraído bruto
- texto limpio
- señales detectadas
- clasificación de municipio
- clasificación de categoría
- clasificación de subtipo
- explicación resumida
- puntuaciones de confianza
- motivos de revisión si procede

========================================
5. PRINCIPIOS CLAVE
========================================

- nunca inventar datos
- no confiar solo en el nombre del archivo
- usar el nombre de carpeta como pista, no como verdad absoluta
- combinar reglas, heurísticas y LLM
- el LLM nunca debe ser la única fuente de decisión
- cada decisión debe poder auditarse
- si la confianza es baja, marcar revisión

========================================
6. MODELO DE DECISIÓN HÍBRIDO
========================================

Quiero una clasificación por capas:

Capa 1. señales estructurales
- carpeta raíz
- subcarpeta
- nombre del lote
- tipo de archivo
- cercanía de archivos

Capa 2. señales textuales deterministas
- palabras clave
- patrones de fecha
- patrones de agenda
- patrones de receta
- patrones de libro
- entidades conocidas
- encabezados típicos

Capa 3. heurísticas de consistencia
- contenido contradice carpeta
- varias señales apuntan a otra categoría
- fecha muy clara pero tipo de texto dudoso

Capa 4. LLM como respaldo
- resolver ambigüedades
- resumir razonamiento
- proponer clasificación
- nunca decidir en solitario sin combinar con señales previas

========================================
7. AGRUPACIÓN DE ARCHIVOS
========================================

Quiero una capa de grouping independiente.

Implementa estrategias como:

1. FolderGroupingStrategy
- si hay carpeta propia de artículo, agrupar por carpeta

2. DocumentPlusImagesGroupingStrategy
- si hay un DOCX o PDF y varias imágenes próximas, asociarlas

3. LooseFilesHeuristicGroupingStrategy
- si hay archivos sueltos, agrupar por similitud de nombre, fecha, proximidad y señales internas

4. SinglePosterGroupingStrategy
- si solo hay una imagen tipo cartel, crear un candidato único

Cada estrategia debe devolver:
- candidate_key
- archivos asociados
- confidence
- reasoning
- strategy_name

Quiero un GroupingOrchestrator que pruebe estrategias en orden y combine resultados.

DOCX:

Imagen:
